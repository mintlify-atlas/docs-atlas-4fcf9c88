---
title: "Sessions API"
description: "List, inspect, and kick active connections and sessions for RTSP, RTMP, WebRTC, SRT, HLS, and recordings."
---

The sessions API gives you visibility into all active protocol connections and sessions. For protocols that have a concept of both a TCP connection and a session (like RTSP), both are exposed separately. You can kick any session to forcefully disconnect a client.

<Note>
  The API must be enabled with `api: true` in `mediamtx.yml`. See the [overview](/api/overview) for setup details.
</Note>

---

## RTSP connections

RTSP uses persistent TCP connections. A single connection may carry one or more RTSP sessions. Connections and sessions are tracked separately.

### GET /v3/rtspconns/list

Returns a paginated list of all active RTSP connections.

```bash
curl http://localhost:9997/v3/rtspconns/list
```

**Query parameters**

<ParamField query="page" type="integer" default="0">Page number (zero-indexed).</ParamField>
<ParamField query="itemsPerPage" type="integer" default="100">Items per page.</ParamField>

**Response fields**

<ResponseField name="pageCount" type="integer">Total number of pages.</ResponseField>
<ResponseField name="itemCount" type="integer">Total number of connections.</ResponseField>
<ResponseField name="items" type="object[]">
  Array of `RTSPConn` objects.
  <Expandable title="RTSPConn properties">
    <ResponseField name="id" type="string">UUID of the connection.</ResponseField>
    <ResponseField name="created" type="string">ISO 8601 timestamp of when the connection was established.</ResponseField>
    <ResponseField name="remoteAddr" type="string">Remote IP address and port.</ResponseField>
    <ResponseField name="session" type="string">UUID of the associated RTSP session, if any. `null` if no session is active on this connection.</ResponseField>
    <ResponseField name="tunnel" type="string">Tunnel mode identifier (for RTSP-over-HTTP tunneling).</ResponseField>
    <ResponseField name="inboundBytes" type="integer">Total bytes received from this connection.</ResponseField>
    <ResponseField name="outboundBytes" type="integer">Total bytes sent to this connection.</ResponseField>
  </Expandable>
</ResponseField>

## RTSPS connections and sessions

RTSPS (RTSP over TLS) exposes the same endpoints under `/v3/rtspsconns/` and `/v3/rtspssessions/`. The response schemas are identical to the RTSP equivalents.

| Endpoint | Description |
|----------|-------------|
| `GET /v3/rtspsconns/list` | List all RTSPS connections |
| `GET /v3/rtspsconns/get/{id}` | Get a specific RTSPS connection |
| `GET /v3/rtspssessions/list` | List all RTSPS sessions |
| `GET /v3/rtspssessions/get/{id}` | Get a specific RTSPS session |
| `POST /v3/rtspssessions/kick/{id}` | Kick an RTSPS session |

## HLS muxers

An HLS muxer is created for each path when a client requests it via the HLS server. It stays alive until no clients have accessed it for a configurable duration (`hlsMuxerCloseAfter`).

### GET /v3/hlsmuxers/list

Returns a paginated list of all active HLS muxers.

```bash
curl http://localhost:9997/v3/hlsmuxers/list
```

**Query parameters**

<ParamField query="page" type="integer" default="0">Page number (zero-indexed).</ParamField>
<ParamField query="itemsPerPage" type="integer" default="100">Items per page.</ParamField>

**Response fields**

## Recordings

The recordings API lets you list recorded segments organized by path, retrieve segment lists for a specific path, and delete individual segments.

<Note>
  Recording must be enabled per path with `record: true` in the path configuration. See [Configuration API](/api/configuration) for details.
</Note>

### GET /v3/recordings/list

Returns a paginated list of all recordings, grouped by path.

```bash
curl http://localhost:9997/v3/recordings/list
```

**Query parameters**

<ParamField query="page" type="integer" default="0">Page number (zero-indexed).</ParamField>
<ParamField query="itemsPerPage" type="integer" default="100">Items per page.</ParamField>

**Response fields**

<ResponseField name="pageCount" type="integer">Total number of pages.</ResponseField>
<ResponseField name="itemCount" type="integer">Total number of recorded paths.</ResponseField>
<ResponseField name="items" type="object[]">

  <Expandable title="Recording properties">

      <Expandable title="RecordingSegment properties">

      </Expandable>

  </Expandable>

### GET /v3/recordings/get/{name}

Returns all recording segments for a specific path.

```bash
curl http://localhost:9997/v3/recordings/get/mystream
```

<ParamField path="name" type="string" required>The path name.</ParamField>

**Response** — a `Recording` object with `name` and `segments` fields (same structure as above).

---

### DELETE /v3/recordings/deletesegment

Deletes a specific recording segment identified by path name and start time.

```bash
curl -X DELETE \
  "http://localhost:9997/v3/recordings/deletesegment?path=mystream&start=2024-11-01T00:00:00Z"
```

**Query parameters**

<ParamField query="path" type="string" required>
  The path name the segment belongs to.
</ParamField>

<ParamField query="start" type="string" required>
  The start timestamp of the segment to delete, as returned in the `segments[].start` field.
</ParamField>

**Response**

```json
{ "status": "ok" }
```

<Warning>
  Deleting a recording segment is permanent and cannot be undone.
</Warning>

## Kicking all sessions on a path

To disconnect all clients reading a specific path, list the sessions, filter by path, and kick each one:

```bash
# Get all RTSP session IDs on "mystream"
IDS=$(curl -s http://localhost:9997/v3/rtspsessions/list \
  | jq -r '.items[] | select(.path == "mystream") | .id')

# Kick each session
for ID in $IDS; do
  curl -sX POST "http://localhost:9997/v3/rtspsessions/kick/$ID"
done
```

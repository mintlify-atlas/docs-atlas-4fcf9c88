MediaMTX is a ready-to-use, zero-dependency real-time media server and proxy. Publish, read, record, and proxy live video and audio streams using any combination of SRT, WebRTC, RTSP, RTMP, HLS, MPEG-TS, and RTP — with automatic protocol conversion between any supported format.

## Get started

<Steps>
  <Step title="Install MediaMTX">
    Download the binary for your platform or pull the Docker image:
    ```bash
    docker run --rm -it \
      -p 8554:8554 \
      -p 1935:1935 \
      -p 8888:8888 \
      -p 8889:8889 \
      -p 8890:8890/udp \
      -p 8189:8189/udp \
      bluenviron/mediamtx:1
    ```
  </Step>
  <Step title="Publish a stream">
    Use FFmpeg to publish a test stream via RTSP:
    ```bash
    ffmpeg -re -stream_loop -1 -i input.mp4 \
      -c copy -f rtsp rtsp://localhost:8554/mystream
    ```
  </Step>
  <Step title="Read the stream">
    Open the stream in FFmpeg, VLC, or any RTSP-capable player:
    ```bash
    ffplay rtsp://localhost:8554/mystream
    ```
    Or open `http://localhost:8888/mystream` in a browser to read via HLS.
  </Step>
  <Step title="Explore features">
    Configure authentication, recording, hooks, and the Control API in `mediamtx.yml`.
  </Step>
</Steps>

## Supported protocols

| Protocol | Publish | Read |
|----------|---------|------|
| RTSP / RTSPS | ✓ | ✓ |
| RTMP / RTMPS | ✓ | ✓ |
| HLS (LL-HLS, fMP4, MPEG-TS) | ✓ | ✓ |
| WebRTC (WHIP/WHEP) | ✓ | ✓ |
| SRT | ✓ | ✓ |
| MPEG-TS (UDP/Unix socket) | ✓ | — |
| RTP (UDP) | ✓ | — |

<Note>
  MediaMTX is a single binary with no external dependencies. It runs on Linux, Windows, and macOS without installing any runtime or interpreter.
</Note>
